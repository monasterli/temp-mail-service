# Монетизация temp-mail сервиса без зависимости от AdSense

## Главный принцип

Сервис не должен зависеть от одной рекламной сети. В коде используется компонент `AdSlot`, который можно переключать через переменную:

```env
NEXT_PUBLIC_AD_PROVIDER=placeholder
```

Варианты:

```txt
placeholder — заглушки на этапе разработки
direct — прямой баннер / свой рекламодатель
cpa — партнерская ссылка
adsterra — рекламная сеть Adsterra
monetag — Monetag / PropellerAds
propeller — alias для Monetag/PropellerAds
adsense — опционально, только для безопасных SEO-страниц
```

## Стартовая схема для Беларуси

Я бы запускал так:

```txt
1. Direct / CPA banners
2. Privacy/VPN/proxy/password manager партнерки
3. Adsterra или Monetag после модерации
4. Прямые рекламодатели
5. Premium-функции
6. AdSense только если позже появится легальная возможность и подходящие страницы
```

## Где показывать рекламу

Безопасные места:

- верхний баннер на главной;
- боковой баннер на desktop;
- SEO-страницы и статьи;
- экран продления почты;
- footer/banner после основного действия.

Нежелательные места:

- внутри текста письма;
- между строками inbox;
- рядом с приватным содержимым письма;
- в формате, который заставляет пользователя кликать.

## Rewarded-продление

Кнопка `Продлить на 30 минут` показывает рекламный экран и затем вызывает:

```txt
POST /api/mailbox/extend
```

Это лучше монетизировать через CPA/direct/interstitial-форматы, а не через обычный AdSense.

## Что добавить перед реальным запуском

- cookie banner / CMP;
- отдельную страницу Contact;
- реальные Privacy Policy / Terms / Abuse под владельца проекта;
- `ads.txt` для подключенных сетей;
- UTM-метки на CPA-ссылки;
- аналитику кликов по баннерам;
- A/B тест рекламных мест.

## Метрики

Смотреть нужно не только показы рекламы, а всю воронку:

```txt
посетители -> созданные ящики -> полученные письма -> продления -> клики по рекламе -> доход
```

Главные метрики:

- visits/day;
- mailbox_created/day;
- inbound_messages/day;
- extension_click_rate;
- ad_click_rate;
- revenue per 1000 visits;
- повторные пользователи.
